Artifact Overview

The artifact selected for this enhancement is my CS 360 Weight Tracking Android Application, which was created during my mobile architecture and programming course. The application allows users to record their weight, set a target weight, and track their progress over time.

I selected this artifact because it demonstrates how algorithms and data structures can improve the organization, accuracy, and usability of stored information. The weight history provided a practical opportunity to improve how records are represented, sorted, retrieved, and updated.

Algorithms and Data Structures Enhancement

For this enhancement, I changed how the application organizes and displays weight records. Each entry is now represented as a structured WeightRecord object containing a record ID, date, and weight value. This structure allows the application to manage individual entries more accurately.

The application now sorts weight entries from newest to oldest so the most recent information appears first. I updated the database query to retrieve records in the correct order and added a database index to improve retrieval efficiency as more records are saved. I also corrected the edit feature so it updates the selected record instead of inserting a duplicate entry.

Skills Demonstrated

This enhancement demonstrates my ability to use structured data, sorting logic, database queries, indexing, and record-management algorithms to improve an application. It also demonstrates my ability to identify problems in existing logic and make changes that improve accuracy and performance.

Course Outcome

This enhancement supports the course outcome of designing and evaluating computing solutions using algorithmic principles, computer science practices, and appropriate standards. The structured record model, sorting process, and more efficient retrieval method make the application easier to use while keeping the weight history organized and accurate.

Reflection

While completing this enhancement, I learned that data must be organized correctly before it can be processed and displayed reliably. One challenge was keeping the database query, in-memory list, dashboard, and record-editing feature synchronized after users added, edited, or deleted information.
