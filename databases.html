Artifact Overview

The artifact selected for this enhancement is my CS 360 Weight Tracking Android Application, which was originally created during my mobile architecture and programming course. The application allows users to create an account, record weight entries, set a target weight, and track their progress.

I selected this artifact because it demonstrates how database design affects the organization, accuracy, reliability, and security of an application. The project provided an opportunity to improve how account information, weight records, and goal data are stored and protected.

Database Enhancement

Before this enhancement, the application stored account information and target-weight data together in the users table. I improved the database design by separating the information into clearer tables, including a users table for account information, a weight records table for saved entries, and a goals table for target-weight data.

Each weight record and goal is connected to the correct user, which helps prevent information from different accounts from being mixed together. I also replaced plain-text password storage with salted password hashing, which provides stronger protection for user login information.

Database migration support was also added so existing users and records can continue to work after the database structure is updated. The sorting and indexing improvements from the algorithms enhancement remain part of the final application.

Skills Demonstrated

This enhancement demonstrates my skills in relational database design, SQLite development, table organization, data relationships, database migration, authentication, and secure password storage. It also demonstrates my ability to improve an existing database without removing the functionality that was completed during previous enhancements.

Course Outcome

This enhancement supports the course outcome of developing a security mindset that anticipates possible weaknesses, protects private information, and reduces design flaws. Separating the data into organized tables improves reliability, while salted password hashing reduces the risk created by storing passwords as readable text.

Reflection

While completing this enhancement, I learned that database organization and security affect every part of an application. One challenge was changing how accounts and goals were stored while keeping the login, dashboard, weight history, and goal-setting features working correctly. This process improved my understanding of database relationships, migration, and secure data handling.
